# Release Notes Template

Use this template as the stable structure for GitHub release notes.

## Summary

One short paragraph that says what this release means in practical terms.

## Since

- Previous tag: `vA.B.C` or `none`
- Release date: `YYYY-MM-DD`

## Highlights

- Highlight 1
- Highlight 2
- Highlight 3

## What Changed

### Added

- Added item

### Changed

- Changed item

### Fixed

- Fixed item

## Validation

- `./scripts/check.sh`
- wheel install verification
- any extra repo-specific verification

## Notes

- known limits that still remain
- what is intentionally out of scope for this release

## Next

- the most likely next milestone or follow-up focus

## Output Format

- do not include a top-level title; GitHub release title is filled separately
- when drafting release notes in chat, return the final body inside a fenced `md` code block
