# Intent 路线图

中文 | [English](../EN/roadmap.md)

## 当前总原则

- 当前实现以 [CLI 统一设计文档](cli.md) 为准
- 这份路线图描述的是阶段顺序，不是长期不变的理念
- 路线图可以随着 dogfooding 和真实使用反馈调整

## 阶段一：把本地语义层做扎实

当前重点：

- 让 CLI 实现和 [cli.md](cli.md) 对齐
- 用真实 agent workflow 验证 `intent / snap / decision` 这组对象是否足够稳定
- 验证中断恢复、续做、长期决策沉淀这些核心场景
- 收敛 inspect、list、show、create、状态流转等基础行为

这一阶段的目标不是“功能尽可能多”，而是“语义层尽可能稳”。

## 阶段二：把 agent 使用方式收敛下来

当本地语义层足够稳定之后，下一步是收敛 agent 如何自然使用它。

重点包括：

- 何时应该创建新对象，何时应沿用已有对象
- 如何让记录成本足够低，而不是变成额外负担
- 如何让 agent 在真实工作中维护语义连续性，而不是只在事后补记
- 如何通过 dogfooding 找到高信号、低噪声的使用方式

这一阶段关注的是 workflow，而不是平台化。

## 阶段三：再考虑远端组织与协作层

更后续的方向包括：

- 共享 timeline
- 跨人和跨 agent 的协作上下文
- 更清晰的展示、检索和回溯视图
- 团队级的交接、review 和决策继承

这些方向很重要，但它们依赖本地语义层先足够稳定。

## 当前非目标

现阶段不优先做：

- 重型项目管理平台
- 完整原始对话归档系统
- 为远端协作提前设计过多复杂机制
- 脱离真实 workflow 的抽象平台化

## 判断路线是否走对

路线是否正确，主要看这些信号：

- CLI 模型是否越来越少需要推翻重来
- agent 是否越来越少依赖临时解释来续做工作
- 人类是否越来越容易看懂当前语义状态
- 新增复杂度是否换来了真实收益，而不是文档和流程负担
