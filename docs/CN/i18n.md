# Intent 文档国际化规范

目的：定义当前 EN/CN 双语文档结构及其维护规则。

这份文档回答：

- 双语文件应该如何组织
- `EN/CN` 命名规则应该如何落地
- EN/CN 文件之间应如何同步
- 这套文档结构后续应如何维护

这份文档不回答：

- CLI contract 细节
- 实现路线图细节
- 发布或社区运营问题

## 目标

- 为核心文档同时提供 EN 和 CN 版本
- 避免双语内容长期漂移
- 控制维护成本

## 当前状态

- 仓库当前使用 EN/CN 双语文档结构
- GitHub 入口优先展示英文 README
- 核心文档已经同时存在于 `docs/EN/` 和 `docs/CN/`
- 根目录 README 结构已经固定为 `README.md` 和 `README.CN.md`

## 核心原则

- 使用对称的 EN/CN 结构
- 避免在同一个文件里做中英混排
- 使用 `README.md` 作为英文 GitHub 入口，`README.CN.md` 作为中文根目录版本

## 目录策略

当前结构：

```text
README.md
README.CN.md

docs/
  README.md
  EN/
    README.md
    glossary.md
    vision.md
    cli.md
    demo.md
    roadmap.md
    i18n.md
  CN/
    README.md
    glossary.md
    vision.md
    cli.md
    demo.md
    roadmap.md
    i18n.md
```

规则：

- `README.md` 是英文 GitHub 展示入口
- `docs/README.md` 是一个轻量的语言选择入口
- 文档主体放在 `docs/EN/` 和 `docs/CN/` 两棵语言树下
- 两棵语言树中的文件名保持一致
- 根目录 README 仍是唯一命名例外：英文使用 `README.md`，中文使用 `README.CN.md`

## 命名规则

- 英文文档使用 `docs/EN/`
- 中文文档使用 `docs/CN/`
- 两棵语言树中的文件名保持一致

示例：

- `docs/EN/vision.md` <-> `docs/CN/vision.md`
- `docs/EN/cli.md` <-> `docs/CN/cli.md`
- `README.md` <-> `README.CN.md`

## 同步规则

- 每对文档都应该在顶部直接链接到对方
- 先更新当前语言版本，再显式同步另一种语言版本
- 必要时可以加入 `Last synced with: <commit>` 说明同步点

## 链接规则

- `README.md` 应链接 `README.CN.md`
- 英文文档默认只链接英文文档
- 中文文档默认只链接中文文档
- 顶部语言切换链接负责跨语言跳转
- `docs/README.md` 继续作为 `docs/` 目录下的语言选择入口

## 编写规则

- CLI 命令、对象 id、flags、JSON 字段名不翻译
- 核心对象术语保持英文：`intent`、`checkpoint`
- 英文写作应以自然英文重写，而不是逐句直译
- 两种语言都采用清晰、中性的开源项目文档语气
- 避免在同一篇文档中使用双语段落

## 当前覆盖范围

目前已在两棵语言树中覆盖的核心文档：

- `README.md` / `README.CN.md`
- `docs/EN/README.md` / `docs/CN/README.md`
- `docs/EN/glossary.md` / `docs/CN/glossary.md`
- `docs/EN/vision.md` / `docs/CN/vision.md`
- `docs/EN/cli.md` / `docs/CN/cli.md`
- `docs/EN/demo.md` / `docs/CN/demo.md`
- `docs/EN/roadmap.md` / `docs/CN/roadmap.md`
- `docs/EN/i18n.md` / `docs/CN/i18n.md`

## 维护流程

- 先更新当前语言版本
- 再显式同步另一种语言版本
- 同语言文档之间默认保持同语言链接
- 跨语言跳转只保留在顶部语言切换和索引入口中
- 结构调整后，检查 `docs/README.md` 是否仍然能正常承担语言选择入口

## 维护策略

- 新的公开文档应尽量在两棵语言树中同时提供
- 如果某一种语言先落地，另一种语言要被显式排入同步计划

## 简短结论

Intent 保持 `README.md` / `README.CN.md` 的根目录结构，并将 `docs/` 组织成镜像的 `EN/` 与 `CN/` 语言树，通过显式同步维护双语一致性。
